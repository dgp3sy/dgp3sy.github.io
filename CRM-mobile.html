The Process

During my time working on this project, I focused on working on the Android Application, which I chose because I wanted to broaden my horizons, as primarily an Apple User. While I helped with the backend of the iOS portion, which was developed using Xamarin.Forms, I focused on the full stack development on the Android Application. This was an incredibly rewarding experience, because it allowed me to understand the full scope of the project, and allowed me to gain insight on how I would stucture the work I needed to complete each sprint. The android development was done using Xamarin Android, because I wanted the users to have a native feel while using the application, in order to grant them primier user experience. The User Interface was designed using AXML, which is android specific, and the backend work was done using C#, which is driven by Xamarin. The use of Xamarin allowed both teams to share backend code of the mobile application throughout the entire process. In addition to the full stack development, we also used a SQL database to house data that persists throughout the application. This allowed me to expand upon the SQL skills that I have learned in classes at school. In addition, I worked on the API calls to pull JSON information that would be displayed, which I found extremely interesting since that allowed the dynamic to be functional even as the business developed after our internship was over. We also implemented a number of security measures in order to protect the information displayed on the application. Throughout the entire process, I had fun gaining a new skill set and I earned an appreciation for all the work I accomplished throughout the summer. This project was much more rewarding than simple data entry or minor bug fixes that some of my peers from school worked on because it allowed me to earn hands-on experience into the development of a real project that creates value to the company.

The Final Product

The final product shown below uses testing data, therefore none of the information below relates to an actual person, but is rather used to show functionality of the interface. The target user of the end product was the executives at the company, who would use the mobile application while interacting with clients face to face. Both the target user and the intended use of the application was extremely important and guided the design of the User Interface of the application. When the application is loaded, the user is directed to the home screen, where they are able to choose next where to navigate using the bottom navigation bar. Each click on the navigation bar will display a fragment of User Interface, which allows the application to load the screen faster than a new page.

The user can then search for advisor information using first name, last name, or a unique identifying code. This search function is handled using auto-complete and will query the underlying database accordingly to show search results of that text and similar text using wildcard search. The search will fill a list view with advisor information, including name and picture, which are the two essential points of data that the user would need to pull up the information of the individual they are networking with. Once the user selects an advisor, they are able to see additional information about the performance of the individual within the firm, as well as contact information. The user can see additional categories of information using the dropdown menu that is placed directly below the header. The application was designed to display the most pertinent data to the user first, which would enhance the quality of interactions between the user and the clients. The user has the option to write notes about the individual or add them to a specific favorites list in order to quickly return to the user later.
